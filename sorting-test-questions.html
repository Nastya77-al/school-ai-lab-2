Тест: Алгоритми сортування

1. Яка часова складність алгоритму Bubble Sort у найгіршому випадку?

2. Який алгоритм використовує принцип "розділяй і володарюй"?

3. Скільки порівнянь виконає Selection Sort для масиву з 5 елементів?

4. Який алгоритм є стабільним?

5. Що відбудеться після першого проходу Bubble Sort?

arr = [5, 2, 8, 1, 9]

6. Яка просторова складність Merge Sort?

7. Який алгоритм виконує мінімальну кількість обмінів?

8. Для якого типу даних Bubble Sort працює найшвидше?

9. Яка основна перевага Merge Sort над Bubble Sort?

10. Який з алгоритмів НЕ є адаптивним?

Пояснення відповідей:

Питання 1: Bubble Sort має складність O(n²), оскільки використовує два вкладені цикли.
Питання 2: Merge Sort використовує принцип "розділяй і володарюй", розбиваючи масив на менші частини.
Питання 3: Selection Sort виконує (n-1)+(n-2)+...+1 = n(n-1)/2 порівнянь. Для n=5: 4+3+2+1=10.
Питання 4: Merge Sort є стабільним - зберігає порядок однакових елементів.
Питання 5: Після першого проходу найбільший елемент (9) опиниться в кінці: [2, 5, 1, 8, 9].
Питання 6: Merge Sort потребує O(n) додаткової пам'яті для тимчасових масивів.
Питання 7: Selection Sort виконує максимум n-1 обмінів (по одному на кожну ітерацію).
Питання 8: Оптимізований Bubble Sort працює за O(n) на вже відсортованому масиві.
Питання 9: Merge Sort завжди працює за O(n log n), на відміну від O(n²) у Bubble Sort.
Питання 10: Selection Sort не адаптивний - завжди виконує однакову кількість операцій.
